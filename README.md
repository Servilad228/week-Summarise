# Telegram Weekly Summarizer (Недельный Саммаризатор Telegram-чатов)

Этот проект представляет собой скрипт на Python, который автоматически собирает историю сообщений из определенного Telegram-чата за прошедшую неделю, отправляет этот текст нейросети через OpenRouter (по умолчанию используется `google/gemini-2.5-flash` благодаря ее огромному контекстному окну и высокой скорости работы), генерирует структурированный отчет происходящего в группе на русском языке и отправляет результат в указанные чаты (по умолчанию — в ваши «Сохраненные сообщения»).

Проект полностью упакован в Docker для удобного развертывания на сервере с автоматическим перезапуском при сбоях или перезагрузке системы. Состояние (дата последнего запуска) сохраняется в локальный JSON-файл, поэтому при перезапусках счетчик времени не сбрасывается.

---

## 🛠️ Возможности

1. **Два режима работы (задаются в `.env`)**:
   - `interval` (интервальный): Скрипт срабатывает раз в N дней (по умолчанию 7) начиная с даты последнего успешного запуска.
   - `scheduled` (по расписанию): Скрипт срабатывает в строго определенный день недели и время (например, каждое воскресенье в 20:00) в часовом поясе, указанном пользователем.
2. **Отказоустойчивость**: Если сервер был выключен во время запланированного запуска по расписанию, при последующем включении скрипт мгновенно поймет, что пропустил запуск (сравнив последнюю дату запуска со временем расписания), сформирует отчет и отправит его.
3. **Безопасное хранение сессии**: Файл сессии Telethon и файл состояния сохраняются в директории `data`, которая вынесена в Volume (постоянное хранилище).
4. **Гибкая отправка**: Отчет может отправляться в один или несколько чатов/каналов/пользователей через запятую (например, в «Сохраненные сообщения» или непосредственно в тот же чат).

## 📋 Подготовка и регистрация ключей

### 1. Получение Telegram API credentials
Telethon использует официальное API Telegram (как клиентское приложение):
1. Перейдите на сайт [my.telegram.org](https://my.telegram.org/).
2. Авторизуйтесь под своим номером телефона.
3. Перейдите в раздел **API development tools**.
4. Создайте новое приложение (заполните любое имя и короткое имя).
5. Скопируйте **App api_id** и **App api_hash**.

### 2. Получение ключа OpenRouter
1. Перейдите на [openrouter.ai](https://openrouter.ai/).
2. Создайте аккаунт и перейдите в настройки для создания API Key.
3. Пополните баланс на небольшую сумму (модели вроде Gemini 2.5 Flash стоят десятые доли цента за запрос, поэтому $1-2 хватит на месяцы работы).

## ⚙️ Установка и настройка

1. Клонируйте этот репозиторий на ваш сервер или локальный ПК.
2. Создайте файл конфигурации `.env` на основе шаблона:
   ```bash
   cp .env.example .env
   ```
3. Откройте `.env` и заполните обязательные параметры:
   - `TELEGRAM_API_ID` — ваш API ID.
   - `TELEGRAM_API_HASH` — ваш API Hash.
   - `TELEGRAM_CHAT_ID` — ID или юзернейм чата, откуда скачивать сообщения (например, `@my_group` или `-100123456789`). Чтобы узнать ID чата, можно переслать любое сообщение из него в бота вроде `@GetIDsBot`.
   - `OPENROUTER_API_KEY` — ваш API-ключ от OpenRouter.
   - Настройте `RUN_MODE`, `TIMEZONE` и параметры расписания.

## 🚀 Первоначальный запуск и авторизация (Важно)

При первом запуске Telegram потребует ввести одноразовый код авторизации (который придет вам в приложении Telegram) и пароль двухфакторной аутентификации (если он установлен). **В фоновом режиме Docker это сделать нельзя**, поэтому первый запуск мы делаем интерактивным:

Выполните команду:
```bash
docker compose run --rm summarizer --login
```

Скрипт попросит ввести:
1. Номер телефона (в международном формате, например, `+79991234567`).
2. Код подтверждения из Telegram.
3. Пароль двухфакторной аутентификации (если включен).

После успешной авторизации сессия сохранится в файл `data/session.session`, и контейнер завершит работу.

## 🐳 Запуск службы в фоновом режиме

После успешной авторизации запустите сервис в фоновом режиме:
```bash
docker compose up -d
```

Контейнер будет работать постоянно в фоне. Благодаря параметру `restart: unless-stopped`, он автоматически запустится после перезагрузки сервера или при возникновении сбоя.

## 👨‍💻 Полезные команды

### Просмотр логов
```bash
docker compose logs -f
```

### Ручной мгновенный запуск
Если вы хотите сгенерировать отчет прямо сейчас за последние 7 дней (не дожидаясь расписания):
```bash
docker compose run --rm summarizer --run-now
```

Сделать ручной отчет за другое число дней (например, за последние 3 дня):
```bash
docker compose run --rm summarizer --run-now --days 3
```

*Примечание: Ручной запуск обновит дату последнего запуска, поэтому интервал/расписание скорректируются соответственно.*

### Остановка сервиса
```bash
docker compose down
```

## 📂 Структура проекта
- `main.py` — точка входа, обрабатывает аргументы командной строки и инициализирует процессы.
- `config.py` — загрузка и валидация конфигурации из среды и `.env`.
- `telegram_client.py` — авторизация, выгрузка сообщений и отправка готового summary.
- `summary_generator.py` — взаимодействие с OpenRouter API и промпт.
- `scheduler.py` — логика расписаний (интервальное/календарное) и сохранение состояния.
- `data/` — (создается при запуске) папка с файлом сессии и `state.json`. **Важно добавить в бэкап при переносе сервера!**
